Supabase 배포 가이드

실제 DB로 옮겨서 팀 전체가 함께 쓰는 방법. 준호님 기준 약 15분.

이 가이드의 결과물: 팀원 누구나 URL 접속 → 로그인 없이 수정 → 실시간 동기화. URL은 내부 공유용(사내 전용 Vercel 배포 혹은 자체 서버).

🗂 다운로드할 파일 먼저 확인

이 프로젝트의 Download 버튼으로 전체 ZIP을 받은 뒤, 아래 두 SQL 파일이 있는지 확인하세요.

Status(2026) 엑셀 기반 초기 데이터

⚡ 15분 셋업

1

Supabase 가입 & 프로젝트 생성 (3분)

  1. supabase.com → Start your project → GitHub 계정으로 로그인
  2. New Project 클릭
  3. 프로젝트명: resource-hub (아무거나)
  4. Database Password: 랜덤 생성 눌러서 복사 → 1Password/메모 어딘가에 저장 (나중에 쓸 일은 없음)
  5. Region: Northeast Asia (Seoul)
  6. Create new project → 프로비저닝 1~2분 대기
💡 무료 티어로 충분: 프로젝트 2개, DB 500MB, 대역폭 2GB/월. 45명 × 53주 가동률 데이터는 500KB도 안 됩니다.
2

스키마 실행 (2분)

  1. 좌측 메뉴 → SQL Editor → New query
  2. 이 프로젝트 파일 중 supabase/schema.sql 전체 복사 → 에디터에 붙여넣기
  3. 우하단 Run (또는 Cmd+Enter)
  4. "Success. No rows returned" 나오면 OK
확인: 좌측 Table Editor에 teams, users, utilization, pipeline 4개 테이블이 보이면 성공.
3

초기 데이터 투입 (1분)

  1. 같은 SQL Editor에서 New query
  2. supabase/seed.sql 전체 복사 → 붙여넣기 → Run
  3. Table Editor에서 users 열어보면 46명 들어와 있어야 함
4

API 키 확인 (30초)

  1. 좌측 메뉴 → ⚙️ Project Settings → API
  2. 아래 두 값을 복사:
    • Project URL — https://xxxxx.supabase.co
    • anon public key — eyJhbGc... 긴 문자열
5

앱에 연결 (30초)

Resource Hub 열기 → 브라우저 개발자 도구 (F12) → Console 탭 → 아래 2줄 실행:

localStorage.setItem('SUPABASE_URL', 'https://xxxxx.supabase.co');
localStorage.setItem('SUPABASE_ANON_KEY', 'eyJhbGc...');
location.reload();

xxxxx와 eyJ... 부분은 4단계에서 복사한 값으로 교체.

확인: 새로고침 후 좌하단에 "☁ Supabase 연결됨" 초록 배지가 3초간 떠야 함. 콘솔에 [Resource Hub] ✓ 연결 성공: 9팀, 46명, 37건, XXX개 가동률 레코드 메시지.

🌐 팀에 배포하기 (Vercel, 5분)

지금은 준호님 컴퓨터에서만 열리는 상태. 팀이 쓰게 하려면 호스팅이 필요합니다. Vercel이 제일 쉬움.

A

전체 프로젝트를 ZIP으로 받기

이 프로젝트 상단의 Download 버튼 → 전체 폴더 ZIP 다운로드 → 압축 해제

B

Vercel 배포

  1. vercel.com 가입 (GitHub 연동)
  2. Add New → Project → Import (혹은 Drag & Drop)
  3. 압축 푼 폴더를 드래그 (Resource Hub.html, src/, supabase/가 포함된 폴더)
  4. Framework: Other, 나머지 기본값
  5. Deploy
1~2분 뒤 resource-hub-xxx.vercel.app URL이 발급됩니다. 팀에 공유하세요.
C

팀원이 접속했을 때 자동으로 Supabase 연결되게 하기

지금은 각자 브라우저에서 localStorage에 키를 넣어야 함. 배포 시엔 Resource Hub.html 맨 위 <head> 안에 아래 한 줄 추가하면 자동 연결됩니다:

<script>
  window.SUPABASE_URL = 'https://xxxxx.supabase.co';
  window.SUPABASE_ANON_KEY = 'eyJhbGc...';
</script>
⚠ anon key는 공개 키라 브라우저에 노출돼도 괜찮지만, 이 URL을 아는 모든 사람이 데이터 수정 가능합니다. 사내 공유만 하거나 간단한 해시 토큰을 URL에 붙이는 방식을 권장합니다. (예: https://resource-hub.vercel.app?k=pw-ab12cd34 — 맞지 않으면 화면 잠금) 필요하시면 다음 버전에서 추가 가능합니다.

🔒 보안 체크리스트

  • 사내망 전용 배포 (Vercel Pro의 Password Protection, 또는 사내 서버)
  • 고객사명 / MM 금액 민감도 확인 — 필요 시 코드화(A사/B사) 또는 열 암호화
  • Supabase anon key는 읽기·쓰기만 가능하며 스키마 변경은 불가 (RLS로 제한됨)
  • 정기 백업: Supabase는 일 1회 자동 백업. 추가로 앱의 Export → JSON/CSV 월 1회 권장
  • 2~3개월 써보고 로그인 필요하면 Supabase Auth로 팀장 9명만 수정 가능하게 업그레이드

🛠 문제 해결

"Supabase 연결 실패" 빨간 배지가 뜨면

  • 콘솔 에러 메시지 확인. URL 오타 / KEY 오타가 대부분
  • localStorage.clear() 후 다시 입력
  • Supabase 대시보드 → Table Editor에서 테이블이 진짜 있는지 확인

변경사항이 다른 탭에 실시간 반영 안 됨

  • Supabase 대시보드 → Database → Replication에서 테이블들이 publication에 포함돼 있는지 확인
  • schema.sql 마지막 부분의 alter publication ... add table ... 구문이 실행됐는지

퇴사자 처리

설정 → 인원 관리 → 상태를 퇴사로 변경 + 퇴사일 입력. 가동률 데이터는 유지되며 리스트에서 회색으로 표시.
